*静态成员变量只被定义（初始化）一次
*静态成员变量属于类，但不属于对象!
*要分清楚初始化和赋值的区别
 int a;(定义，初始化的值不确定) int a = 0;(在定义时初始化) a = 0;（赋值）
*静态成员函数只能访问静态成员；普通成员函数可以访问静态成员
*静态成员变量不论私有公有初始化方式一样，在类外初始化（加类型 假如 static int a;  在类外int 类名:: a = 0;）
*静态类成员同样遵守private/protect/public的访问规则
*静态变量使用之前必须显示初始化。
*	一、类不是模板类：如果把静态成员变量初始化赋值语句和成员函数定义（除非内联）写在头文件，那么头文件被多次包含就会报错重复定义。
	二、类是模板类：模板类一般不用实现文件，可以理解为类型T的链接性为内部的，因为只编译模板类头文件不会产生类型，因此不必担心多次包头重复定义的问题。
